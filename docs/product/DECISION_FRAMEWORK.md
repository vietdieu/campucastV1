# Product Decision Framework

Every new feature or substantial change must pass this 10-question checklist to be considered for implementation.

---

## 🧐 Decision Checklist

1. **Efficiency**: Does this feature help the operator complete their work faster? (YES/NO)
2. **Click Reduction**: Does this reduce the number of clicks required for the primary workflow? (YES/NO)
3. **Cognitive Load**: Does this reduce the mental effort/memory requirement? (YES/NO)
4. **Time Savings**: Does this directly reduce project completion time? (YES/NO)
5. **Error Reduction**: Does this minimize the potential for user errors? (YES/NO)
6. **Runtime Complexity**: Does this unnecessarily complicate the runtime/infrastructure? (If YES -> FAIL)
7. **UI Complexity**: Does this unnecessarily complicate the interface? (If YES -> FAIL)
8. **Navigation Bloat**: Does this require adding new menus or navigation items? (If YES -> FAIL)
9. **Home Screen Bloat**: Does this crowd the Home screen beyond its purpose? (If YES -> FAIL)
10. **Memory Test**: If this feature were removed, would the operator genuinely miss it/notice its absence? (If NO -> FAIL)
