# CommuteCast Product Blueprint v1.0
*The North Star for Enterprise Commute Intelligence*

## 1. User Personas
| Persona | Primary Need | Context |
| :--- | :--- | :--- |
| **The Executive Commuter** | High-level synthesis of industry news. | 15-20 min drive, hands-free. |
| **The Language Learner** | Bilingual (EN/VI) news for immersion. | Public transport, looking for vocabulary. |
| **The Silver Listener** | Audio-first news consumption (no reading). | Morning routine at home. |
| **The Professional Driver** | Traffic and urgent news updates. | Continuous on-road operation. |

## 2. User Jobs (User Intent)
- "I want to create a morning briefing quickly before I leave."
- "I want to listen to my saved news while driving."
- "I want to adjust the voice tone to be more professional."
- "I want to check if my RSS feeds are working correctly."

## 3. Information Architecture (The 4 Workstations)

### I. Home (Operator Desk)
**Goal**: Operational Status & Continuity.
- **Active Production**: Real-time status of current TTS/AI tasks.
- **Next Briefing**: Scheduled missions for the day.
- **System Health**: Notifications for RSS errors or Cloud status.
- **Continue Work**: Quick access to the last unfinished script.

### II. Create (Production Station)
**Goal**: Focused, Linear Production.
- **Step 1: Source Selection** (RSS, URL, Manual Text).
- **Step 2: Content Intelligence** (AI Summary, Rewrite, Translation).
- **Step 3: Audio Studio** (Voice Selection, Music Mix, Preview).
- **Step 4: Fulfillment** (Export, Share, Archive).

### III. Library (Asset Workspace)
**Goal**: Object-Oriented Management.
- **Missions**: Active production projects.
- **Scripts**: Written news content/summaries.
- **Audio**: Published .wav/.mp3 files.
- **Sources**: Managed RSS feeds and connectors.
- **Templates**: Reusable voice/style presets.

### IV. Settings (Mission Control)
**Goal**: Categorized Configuration.
- **Profile**: Personal info, UI Language, Theme.
- **Playback**: Voice presets, Speed, Background Music.
- **Content**: Summary length, Traffic priorities, Language preferences.
- **Sync**: Cloud persistence, Local storage management.
- **System**: Cache, Versioning, Help/Tutorials.

## 4. Operator Workflow (The Loop)
`Home` → `Continue Work` → `Create Station` → `Synthesis` → `Preview` → `Publish` → `Library` → `Home`

## 5. Definition of Done (Beta Milestone)
- **Time to Knowledge**: New user knows app purpose in <30 seconds.
- **Time to Script**: Create a briefing in <2 minutes.
- **Time to Audio**: Export finished audio in <5 minutes.
