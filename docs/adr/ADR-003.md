# ADR-003: Audio Manifest Specification

## Context
Delivering a single monolithic audio file per briefing limits flexibility (no seeking, no streaming, no on-the-fly language switching).

## Decision
The API will return an `AudioManifest` JSON object containing ordered segment URLs rather than the binary audio file itself.

## Consequences
- Enables client-side audio segment streaming.
- Allows seamless switching of audio languages by just requesting a new manifest.
- Enables efficient caching of individual segments.
