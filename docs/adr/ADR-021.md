# ADR-021: Editorial Policy Engine

## Context
Different times of day or user contexts require different program formats. A morning commute should be fast-paced and urgent, while a weekend recap can be longer and more relaxed.

## Decision
We implement an **Editorial Policy Engine**.
1. **EditorialPolicy**: Defines the sequence of `BeatType`s and constraints (duration, topics).
2. **Contextual Mapping**: The engine selects the appropriate policy based on the `UserContext`.
3. **Immutable Rules**: Policies are treated as immutable configuration that guides the `NarrativeComposer`.

## Consequences
- Centralizes the "Brand Identity" of the AI DJ.
- Makes it easy to add new broadcast formats (e.g., "Weekend Travel", "Breaking News Alert").
- Provides a clear constraint set for the Composer to work within.
