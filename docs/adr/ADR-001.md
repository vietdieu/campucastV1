# ADR-001: Domain First Design

## Context
CommuteCast requires a robust, scalable, and technology-agnostic foundation. Previous iterations suffered from coupling business logic directly with UI/API concerns, making refactoring expensive.

## Decision
We will adhere to the "Domain First" principle:
1. Define Business Logic and Domain Entities.
2. Design API Contracts based on Domain Models.
3. Map Database Schemas from Domain Entities.
4. Implement Logic and UI.

## Consequences
- Domain layer remains pure TypeScript (no frameworks).
- API/DB changes do not ripple into core business logic.
- Increased upfront effort, significantly lower maintenance cost.
