# ADR-020: Narrative First Architecture

## Context
A simple playlist is just a list of items. A narrative is a structured experience with a beginning, middle, and end, linked by transitions and editorial intent. To achieve an "AI DJ" experience, we need to separate content selection (Recommendation) from structural assembly (Narrative).

## Decision
We implement a **Narrative Composer Platform**.
1. **NarrativeBeat**: The atomic unit of a broadcast. It can be a `Story`, but also meta-elements like `Opening`, `Transition`, or `Insight`.
2. **Structural Composition**: The `NarrativeComposer` assembles the sequence of beats *before* any text is generated.
3. **Beat Types**: Formalizing roles like `Editorial`, `Insight`, and `Closing` to allow for varied program structures.

## Consequences
- Decouples "What to play" from "How to sequence it".
- Allows for complex program structures that resemble professional radio or podcasts.
- Enables the Runtime to handle playback as a state machine of "Beats".
