# ADR-030: Persona Before Voice

## Context
With the `Editorial Brain` and `Editorial Planner` established in Sprint #015, the system now knows *what* to say, *when* to say it, and the *intent* behind it. However, to create a true AI Radio Host, we need a consistent, recognizable character to deliver the broadcast.

Historically, audio applications have equated "Persona" with "TTS Voice" (e.g., "Use the British Female Voice"). This is a technical perspective. A true persona is defined by their vocabulary, sentence rhythm, transition habits, editorial bias, and humor. A user remembers *how* the host speaks and frames a story, not just the acoustic properties of the TTS engine.

## Decision
1. **Persona Before Voice**: We define "Persona" as an `Editorial Identity`, completely decoupled from the physical `Performance Director` (which handles TTS voices).
2. **Rich Persona Profiles**: A Persona is an immutable configuration that defines:
   - `vocabulary` (e.g., "Thị trường ghi nhận..." vs "Có một tin thú vị...")
   - `formality` and `humor`
   - `sentenceRhythm` (e.g., short/punchy vs long/flowing)
   - `transitionStyle` and `catchphrases`
   - `openingStyle` and `closingStyle`
3. **Dedicated Persona Engines**: We will implement specific engines to enforce this identity:
   - `Vocabulary Engine`: Adapts word choices.
   - `Transition Engine`: Selects contextually appropriate, persona-specific transitions.
   - `Editorial Signature`: Manages signature openings and closings.
   - `Persona Resolver`: Dynamically selects the best persona based on the user's context (e.g., Morning commute -> Morning Host).
4. **Frozen Runtime**: The Persona Platform will sit strictly above the Runtime Core and Performance Director. The Runtime remains completely unaware of Personas.

## Consequences
- The TTS layer becomes a commoditized implementation detail. We can swap TTS providers without changing the AI Host's character.
- The AI Host gains true consistency and personality, fostering parasocial relationships and improving listener retention.
- The architecture is strictly decoupled: Content Intent (Editorial Brain) -> Character (Persona Platform) -> Physical Voice (Performance Director) -> Execution (Runtime).
