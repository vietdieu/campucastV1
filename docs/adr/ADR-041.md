# ADR-041: Production Studio Pipeline Decoupling & Stage Isolation

## Context
The CommuteCast Production Studio consists of multiple steps: News Collection (Stage 1), Editorial Draft (Stage 2), Speech Preparation (Stage 3), and Audio Synthesis (Stage 4). 
Historically, these stages were bound together in a monolithic React hook closure (`handleGenerateBriefing` in `useBriefingGeneration.ts`). 
This coupling introduced:
1. Implicit dependencies on fragile UI components and input text fields.
2. Direct, unvalidated transitions (Stage 2 directly invoking Stage 3 & 4), which prevented stage-by-stage retries, resuming, or standalone testing.
3. Race conditions and duplicated states (e.g. active drafts versus inline text areas).

To elevate the pipeline to an enterprise-grade standard, we must isolate each stage, define strict input/output contracts, and manage execution through a dedicated Orchestrator.

## Decision
We will refactor the Production Studio pipeline to enforce:
1. **Isolated Stage Services**: Every stage is a self-contained service that acts on standard inputs and returns well-typed immutable outputs.
2. **Immutable Stage Outputs**: Define the following immutable artifacts:
   - `ResearchPackage` (Stage 1 Output / Stage 2 Input)
   - `EditorialDraft` (Stage 2 Output / Stage 3 Input)
   - `SpeechTimeline` (Stage 3 Output / Stage 4 Input)
   - `AudioAssembly` (Stage 4 Output)
3. **StageResult Wrap**: Every stage execution is reported using a standard `StageResult<T>` wrapper, offering `idle`, `loading`, `success`, and `error` flags.
4. **State Isolation**: React components must only bind as consumers to the results of these stages and are strictly forbidden from modifying stage execution records or mutating pipeline state directly.
5. **Stage 2 Isolation**: Refactor Stage 2 (Editorial Draft Engine) to consume `ResearchPackage` and produce `EditorialDraft` with zero secondary coupling.

## Consequences
- **Robust Failure Resilience**: If Audio Synthesis (Stage 4) fails, the system can retry Stage 4 directly without re-triggering Editorial Draft Generation (Stage 2), conserving AI credit quota.
- **Enhanced Testability**: Individual stage processors can be tested as pure/isolated services with mock input/output vectors.
- **Maintainability**: Clear and standardized code structure with clean separation of concerns.
- **Complexity**: Introduction of intermediate contracts (`ResearchPackage`, `EditorialDraft`) requires more formal type definitions and state orchestration management.
