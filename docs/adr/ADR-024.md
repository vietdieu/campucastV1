# ADR-024: Performance Manifest v3 Contract

## Context
To decouple `PerformanceDirector` from `Runtime`, we need a strict execution contract. The `PerformanceManifest` serves as this contract, defining the exact instruction set the runtime must execute.

## Decision
We implement **Performance Manifest v3**.
1. **Beat-based Instruction Set**: Mirroring the narrative beat structure with exhaustive performance instructions.
2. **PerformanceProfile Aggregate**: Consolidated data for voice, prosody, emotion, timing, and music cues.
3. **Implicit Execution**: The Runtime is not "intelligent"; it purely executes the manifest.

## Consequences
- Enables independent development and testing of `PerformanceDirector` and `Runtime`.
- Improves the reliability of the audio playback engine.
- Sets a clear contract for future enhancements (e.g., real-time ducking, sound effects).
