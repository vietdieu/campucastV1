# ADR-002: Generate Once, Play Many

## Context
LLM inference is the most expensive part of our pipeline. Regenerating summaries for every user request is financially unsustainable and increases latency.

## Decision
We will adopt a "Generate Once, Play Many" architecture:
1. Briefing is generated once per RSS update.
2. Structured JSON (Briefing + Segments) is stored.
3. Audio is synthesized on-demand (or pre-warmed) based on the structured content.

## Consequences
- Cost reduction by orders of magnitude.
- Instant language switching.
- Briefing data becomes a reusable asset for multiple projections.
