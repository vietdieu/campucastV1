# ADR-005: Persistence Boundary

## Context
The Domain layer must remain technology-agnostic. Tight coupling to persistence frameworks (Drizzle/Postgres) complicates testing, maintenance, and future migrations.

## Decision
Enforce a strict Persistence Boundary:
1. Domain Entities/Aggregates reside purely in `src/domain/`.
2. Infrastructure Mappers convert between Domain Entities and Persistence Records (`src/infrastructure/db/mappers/`).
3. Repositories communicate *only* via Domain Models; they *never* leak database-specific types (e.g., Drizzle/Postgres types) into the Domain.

## Consequences
- The database schema can evolve independently of business logic.
- Domain logic is testable without a database.
- Future database engine changes only require new infrastructure adapters.
