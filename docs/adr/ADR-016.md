# ADR-016: User Intelligence as the Heart of Personalization

## Context
While the Memory Platform (Sprint #007) provides the "Source of Truth" for raw data (history, events, preferences), it does not provide *intelligence*. A Recommendation Engine needs a distilled, vector-based understanding of the user and their current context to make accurate content decisions. Without a dedicated Intelligence layer, recommendation logic becomes cluttered with data processing, history analysis, and context resolution.

## Decision
We establish the **User Intelligence Platform** as a specialized layer between raw Memory and the Recommendation Engine.
1. **Interest Vectorization**: Raw listening history and feedback events are distilled into a dynamic `InterestVector` (e.g., Technology: 0.94, Sports: 0.12).
2. **Context Resolution**: The system resolves the user's current environment (`Morning Commute`, `Deep Night`, `Work Focus`) based on time, location, and device state.
3. **Behavioral Archetypes**: Users are categorized into archetypes (e.g., `Completionist`, `Fast Skipper`, `Deep Listener`) to tune playback duration and summary depth.
4. **Decoupled Analysis**: The Intelligence Platform computes these signals asynchronously or on-demand, providing a "Intelligence Snapshot" to the Recommendation Engine.

## Consequences
- The Recommendation Engine (Sprint #010) becomes a lightweight ranker rather than a heavy data processor.
- Contextual personalization (e.g., changing voice speed for morning vs. night) is handled centrally.
- We can evolve the "Intelligence" (how we calculate interest scores) without changing the database schema or the recommendation ranking logic.
