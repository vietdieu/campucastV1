# ADR-013: AI Vendor Independence

## Context
As CommuteCast integrates more advanced AI capabilities, relying directly on a single vendor's SDK (e.g., Google's Gemini SDK) within the core domain and application layers creates technical debt and vendor lock-in. Models evolve rapidly, and different tasks may require different models (e.g., Gemini for speed, GPT-4 for complex reasoning, Claude for large context, or open-source models for cost).

## Decision
We enforce strict AI Vendor Independence across the platform:
1. **No Direct SDK Dependency**: No service in the Domain or Pipeline layers is allowed to depend directly on any LLM provider's SDK or client library.
2. **LLM Platform Abstraction**: All AI generation must be routed through the `LLMPlatform` interface.
3. **Adapter Pattern**: Specific models must be implemented as Adapters (e.g., `GeminiAdapter`, `MockAdapter`) that conform to a common `LLMProvider` interface.
4. **Capability-Based Routing**: Models are selected by the `ModelRouter` based on declared capabilities (`CapabilityRegistry`) rather than hardcoded names.

## Consequences
- The core pipeline is completely insulated from changes in external AI APIs.
- We can seamlessly hot-swap models or providers based on cost, latency, or feature availability without changing business logic.
- Promotes a unified approach to output validation, safety, caching, and cost telemetry across any model.
