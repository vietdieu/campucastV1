# ADR-006: Deterministic Briefing Generation Pipeline

## Context
Regenerating summaries for identical content is costly and inefficient. The generation process must be reproducible, robust, and capable of identifying redundant requests to minimize LLM usage.

## Decision
We will implement a Deterministic Briefing Generation Pipeline:
1. **Fingerprinting**: Generate hashes (`SHA-256`) based on normalized content (title, link, date, body) excluding non-deterministic metadata.
2. **Pipeline Flow**: RSS -> Parser -> Normalizer -> Fingerprint Check -> Summarize -> Assemble -> Save.
3. **Immutability**: Once saved, a Briefing aggregate is considered immutable.
4. **Error Handling**: Any failure in the pipeline (especially summarization) prevents data persistence to ensure integrity.

## Consequences
- Significant reduction in LLM costs and latency for redundant RSS feeds.
- Pipeline becomes deterministic and easy to debug.
- Data integrity is maintained via transactional safety (no partial data saved).
