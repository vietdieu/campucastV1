# ADR-025: Intent-Based Performance Architecture

## Context
Decoupling `Narrative` from `Runtime` is necessary, but the `Performance Manifest v2` still contained implementation details (`VoiceProfile`, `ProsodyProfile`, etc.) which tightly coupled the `PerformanceDirector` to the audio engine implementation.

## Decision
We implement **Performance Intent Architecture**.
1. **PerformanceIntent**: The manifest now contains an `Intent` (e.g., 'EnergeticMorningHost') instead of a concrete `PerformanceProfile`.
2. **Resolution Layer**: The `PerformanceDirector` acts as the resolution layer, mapping `PerformanceIntent` to a concrete `PerformanceProfile` at execution time.
3. **Execution Contract v3**: The `PerformanceManifest` is now a contract of *intent*, not *implementation*.

## Consequences
- Decouples audio aesthetics from playback logic.
- Enables hot-swapping voices/styles without regenerating the manifest.
- Allows for persona-based host styles ('AI Host Persona Platform').
