# ADR-028: Editorial Intelligence First

## Context
With the successful execution-freeze and certification of the Runtime Core (v4.15.0-RC), CommuteCast has achieved full "Platform Maturity." Technical execution is no longer the bottleneck; the strategic differentiator of the product now depends entirely on *how well* the AI edits, tells stories, and hosts the broadcast program.

To avoid technical and architectural bloat in the lower execution layers, we must establish a clear boundary where all future investment moves from "Platform/Engineering" to "Editorial/Product Intelligence."

## Decision
Establish the **Editorial Intelligence First** paradigm across all system development:
1. **Scope Freeze of Runtime Core**: No new features or changes are allowed in `TimelineScheduler`, `RuntimeSnapshot`, `TimelineEvent`, `RuntimeCommand`, `ExecutionTimeline`, `Plugin Lifecycle`, and `Event Bus Contract`. All future requirements must be addressed in upper layers (e.g., via Plugins or the Editorial Layer).
2. **Upgrade to Editorial Intelligence Platform**: Reframe the Persona and Host layers into a unified Editorial Platform consisting of:
   - **Persona Profiles**: Defining host character, vocabulary, energy, transition habits, and tone.
   - **Editorial Styles**: Standardizing journalistic constraints (e.g., Bloomberg-brief, NPR-narrative, DJ-conversational).
   - **Audience Models & Taste Graphs**: Mapping user demographics, temporal contexts, and behavioral signals (e.g., Morning commute vs. Evening unwind) to concrete experience rules.
   - **Narrative Selection & Role Play**: Restructuring the Recommendation system from flat ranking to dynamic narrative role assignments (e.g., Intro, Outro, Hook, Supporting stories).
3. **Product-Driven Metrics (KPIs)**: Standardize all primary dashboards around Product metrics (Session Completion, Return Rate, Skip Rate, Listener Minutes) while lower-level KPIs (Startup Latency, Gap Duration) are demoted to system Service Level Objectives (SLOs).

## Consequences
- Protects the highly optimized Runtime Core from regression and technical debt.
- Aligns all engineering focus on delivering maximum product ROI and editorial value to the user.
- Directs future development sequentially: **Editorial Intelligence (Persona + Style) ➔ Broadcast Director (Executive Producer) ➔ AI DJ Experience ➔ Personalization & Taste Graph Loop**.
