# ADR-040: Experience Platform UX Operating System

## Status
Proposed (2026-07-04)

## Context
Following the stabilization and freeze of the core execution runtime (Runtime Core 4.16.0), the platform has reached high technical maturity. However, user-facing complexity has increased exponentially due to the proliferation of features (RSS Studio, LLM Prompting, Personal Memory, Audio Synthesis configs, Play Queue, Stats, etc.). To demonstrate genuine product impact and retain user attention, we must transition the app's presentation architecture from feature-centric tabs to task-centric workstations.

## Decision
1. **UX Operating System Refactoring**: We restructure the interface information architecture into a 6-tier Workspace layout:
   - `home`: Workstation launchpad.
   - `create`: Unified feed and narration creation desk.
   - `library`: Media library, smart play queue, and podcast publisher.
   - `aihost`: Broadcast persona controls and active Knowledge Snapshot inspector.
   - `analytics`: Performance indicators, listener statistics, and product impact lineage reports.
   - `settings`: Symmetrical preference toggles and hardware limits.
2. **Tab State Mapping**: All previous tabs are mapped cleanly to these 6 high-level views to preserve backwards compatibility and existing capabilities.
3. **Snapshot Visibility**: We elevate the WP-1 Snapshot Intelligence Contract from a hidden backend entity to a prominent, read-only "Knowledge Governance Terminal" inside the `aihost` dashboard. This lets operators see live hypotheses, validation results, and rollout status.

## Rationale
Reorganizing around tasks rather than underlying technologies makes the platform instantly understandable and increases efficiency. By putting the most important info in front of the user (e.g. current compilation draft on Home, Snapshot status on AI Host, Product Scorecard on Analytics), we optimize user workflow and reinforce the evidence-driven character of Baseline 2.0.

## Consequences
- **Positive**: Exceptional visual clarity, higher task completion rate, full alignment with the "Measure & Explain" principles of Baseline 2.0, non-destructive integration.
- **Negative**: Reordering sidebar items might cause brief disorientation for legacy users.
- **Mitigation**: Use highly descriptive labels and bilingual subtitles to make the new navigation self-explanatory.
