# ADR-026: Experience-Driven Broadcast Architecture

## Context
Decoupling `Performance` from `Runtime` is achieved via `PerformanceIntent` (ADR-025), but the system still lacks a holistic "experience" layer. The `PerformanceDirector` still implicitly decides *how* an experience should be felt.

## Decision
Implement an **Experience Layer** comprising:
1. **Experience aggregate**: A holistic definition of a broadcast style (Persona, Music, Environment).
2. **ExperienceRegistry**: A central store of available "Experience" profiles.
3. **ExperienceDirector**: The top-level director that maps context (Morning, Rain, Traffic) to an `Experience`.
4. **PerformanceDirector**: Refactored to act strictly as a Resolver mapping `PerformanceIntent` (derived from `Experience`) to a concrete `PerformanceProfile`.

## Consequences
- Elevates the system to be "Experience-driven" rather than "Command-driven".
- Enables the future "AI Persona Marketplace".
- Provides a clean separation between "What the user feels" (Experience) and "How it is performed" (Performance).
