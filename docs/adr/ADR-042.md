# ADR-042: Stage 4 Audio Rendering Engine Isolation & Resilience Enhancement

## Context
In Sprint STU-110, we isolated Stage 2 and Stage 3 of the Production Studio pipeline. However, Stage 4 (Audio Rendering Engine / voice synthesis) remained coupled inside a local loop in `useBriefingGeneration.ts`. This layout lacked structural flexibility, preventing features like resume rendering, robust cache matching, strict cancellation via AbortSignals, loudness normalization records, and proper verification checksum validations. 

To complete our enterprise-grade pipeline architecture, Stage 4 must be decoupled into a pure service layer that accepts ONLY a `SpeechPackage`, manages all synthesis dependencies cleanly, and returns an immutable `AudioArtifact`.

## Decision
We refactored Stage 4 of the Production Studio pipeline to enforce the following:
1. **Pristine Stage 4 Service**: Defined `renderAudio` in `/src/services/productionPipeline.ts` which accepts ONLY a `SpeechPackage` and custom primitive parameters. It is strictly forbidden from accessing `Draft`, `RSS`, `Textarea`, `React Form`, or `UI State`.
2. **AudioArtifact Schema**: Enhanced the output structure from a basic base64 list into an immutable `AudioArtifact` containing:
   - Base64 chunks.
   - Exact listings of successful and failed segments.
   - Fast, collision-resistant rolling DJB2 checksum validation.
   - Comprehensive metadata: duration, sample rate (24kHz), channel count (1 - Mono), bit rate, a `voiceManifest` mapping segment IDs to selected voices, and volume level offset (-14.0 LUFS/dB target).
   - An `isPartial` flag to represent multi-track compilations where some elements failed.
3. **Advanced Resilience & Control Features**:
   - **Cancellation**: Integrated full AbortSignal tracking throughout the sequence and fetch calls to allow instant interruption.
   - **Caching**: Configured a reliable `sessionStorage` in-memory voice-text-emotion cache to skip costly synthesis API calls on identical text segments.
   - **Resume Rendering**: Allowed passing of an `existingArtifact` to skip already-rendered chunks and resume synthesis seamlessly.
   - **Exponential Backoff**: Introduced an automatic retry helper with exponential backoff for high-availability synthesis.

## Consequences
- **Zero API Waste**: The cache and resume rendering mechanics ensure we never make duplicate, expensive TTS API calls for unchanged elements.
- **Flawless User Control**: Users can cancel long rendering sessions instantly without causing resource leaks or zombie network fetches.
- **Enterprise Traceability**: The `AudioArtifact`'s metadata block and DJB2 checksum offer a complete audit trail for delivered audio briefings.
- **Robust Fault Tolerance**: If a single segment fails synthesis, the engine still builds a valid, partial briefing track with the remaining completed components, providing a continuous listening experience.
