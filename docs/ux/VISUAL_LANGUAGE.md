# Visual Language & Spatial Design System

## 1. Visual Hierarchy Foundation
- **Visual Weight**: Define prominence (Hero > Primary Action > Task > Secondary > Stats > Deco).
- **Surface Hierarchy**: Define layers (App Background -> Workspace -> Panel -> Control).
- **Attention Model**: Direct user focus via contrast and placement.

## 2. Spatial Design System
- **Spatial Grammar**: Define layout regions for each workstation type (Home, Studio, Library, Settings).
- **Reading Order**: Consistent content flow (Greeting -> Task -> Action -> Activity -> Insights).
- **Density System**: Standardized spacing, gaps, and margins.

## 3. Visual Language Specifications
- **Color Responsibility**: Semantic mapping for Brand, CTA, Success, Danger, Info, Attention.
- **Border Philosophy**:
  - Border is NOT for layout.
  - Used only for: Input fields, Selectable Cards, Focus states, Danger states.
  - Use surface elevation (shadows/spacing/bg color) for layout separation.
- **Motion Rules**: Consistent transitions for panels, cards, buttons, loading/skeletons.
- **Typography**: Enterprise-grade hierarchy (Display, Heading, Section, Card, Body, Caption, Label).
- **Accessibility**: High contrast standards and consistent states.
- **Theme Mapping**: Consistent behavior across Light, Dark, Soft, and Auto modes.
