# Operator Journeys

This document describes the key user journeys in CommuteCast, focused on a workflow-first, project-centric model.

## 1. Journey: Create Briefing from RSS
1. **Open App**: Land on "Home" (Resume Console).
2. **Start Create**: Click "New Briefing" → "From RSS".
3. **Studio Desk**: Select source, AI drafts script.
4. **Edit**: Review & edit script.
5. **Generate**: Trigger Audio & Video generation.
6. **Publish**: Finalize & Publish to Library.

## 2. Journey: Create Video from Briefing
1. **Resume/Create**: Select existing project or start new from briefing.
2. **Studio Desk**: Open existing Project.
3. **Assets**: AI generates thumbnail/subtitles.
4. **Compile**: Trigger render.
5. **Review/Publish**: Preview in Project view, Publish to Library.

## 3. Journey: Continue Working
1. **Open App**: Home shows "Resume" for last active project.
2. **Resume**: Click "Continue" → Takes user directly to the exact step in Studio Desk (e.g., Edit script).
3. **Working**: Finish editing, generate audio/video.

## 4. Journey: Locate & Edit Existing Project
1. **Library**: Open Library (Unified Asset Manager).
2. **Find**: Use search/filter to find Project.
3. **Manage**: Click Project to open it in Studio Desk for further editing.

## 5. Journey: Export & Share
1. **Project View**: Within Project, go to Output tab.
2. **Publish**: Share directly to social media or download assets.
