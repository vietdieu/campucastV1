# Component Grammar

This document defines the strict component usage rules for CommuteCast.

## 1. Card Types
- **Hero Card**: Focus area for primary tasks.
- **Action Card**: Trigger for specific workflow steps.
- **Information Card**: Display contextual data (e.g., stats).
- **Status Card**: Active task tracking.
- **Summary Card**: Snapshot of project/draft.
- **Danger Card**: Irreversible actions.

## 2. Button Types
- **Primary**: Main action (e.g., "Continue", "Generate").
- **Secondary**: Supporting action (e.g., "Cancel", "Save Draft").
- **Ghost/Quiet**: Low-emphasis actions.
- **Danger**: Destructive actions.

## 3. Panel Types
- **Hero Panel**: Main workspace container.
- **Workflow Panel**: Sequential task steps.
- **Context Panel**: Contextual information.
- **Inspector Panel**: Detailed settings/properties.

## 4. Badge Types
- **Info/Success/Warning/Danger/Neutral**: Semantic status indicators.
