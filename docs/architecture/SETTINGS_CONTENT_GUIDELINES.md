# Settings Content Guidelines

Settings must speak the language of the **Operator**, not the **Developer**.

## 1. Labeling Philosophy
- **Goal over Technology**: Use "Briefing Sources" instead of "RSS Feed Management."
- **Benefit over Parameter**: Use "Clearer Text" instead of "Anti-Aliasing."
- **Action over Noun**: Use "Check for updates" instead of "Update Status."

## 2. The Jargon Translation Table

| Developer Term | Operator Term |
| :--- | :--- |
| RSS Feed | News Source |
| PCM Audio | Broadcast Sound |
| Pitch / Frequency | Voice Tone (High/Low) |
| Latency | Speed |
| Token Limit | Story Length |
| Snapshot | Save Point |
| Cache | Local Storage |

## 3. Help Text
Every section must include a single sentence of **Contextual Help**:
- *Example*: "Voice Tone: Adjust how deep or sharp the broadcast sounds to your ears."
- *Avoid*: "Pitch: Sets the fundamental frequency of the audio output."
