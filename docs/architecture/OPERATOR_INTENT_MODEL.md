# Operator Intent Model

The Operator Intent Model shifts CommuteCast from a "Configuration App" to an "Intent-Driven Platform."

## 1. The Three Layers of Intent

### Layer 1: The Goal ("I want...")
This is the only layer visible to the casual operator. It focuses on human outcomes.
- "I want to hear the news faster."
- "I want a male voice."
- "I want more AI news."
- "I want my data to be private."

### Layer 2: The System Mapping ("The system will...")
The logic layer that translates Goals into multiple parameter changes.
- Goal: "Read faster" -> Action: `Speed +10%`, `Pitch -5%`, `Summary Complexity: Low`.
- Goal: "Personalized" -> Action: `Enable Memory`, `Apply Persona`, `Filter RSS by Keywords`.

### Layer 3: Advanced ("I am an expert...")
The raw parameters for developers and power users.
- `PCM Sample Rate`, `Gemini Temperature`, `RSS Refresh Interval`, `Cache TTL`.

## 2. Intent Taxonomy
Missions and Settings must be categorized by these intents:
1. **Speed & Efficiency**: Time-to-brain optimization.
2. **Atmosphere & Tone**: How the broadcast "feels."
3. **Relevance & Precision**: Content accuracy.
4. **Privacy & Security**: Data control.
5. **System Health**: Reliability and maintenance.

## 3. Dynamic Intent Engines
Future AI Agents will use this model to automatically configure the system based on natural language commands (e.g., "Make it sound more professional").
