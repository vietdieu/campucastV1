# Settings Search & Discovery Model

Users should find settings via **Meaningful Search**, not by memorizing the menu structure.

## 1. Semantic Indexing
Every setting must be tagged with human-centric keywords:
- **Field**: `voiceSpeed`
- **Keywords**: "faster", "slower", "slow down", "reading speed", "tempo".

## 2. Search Results Hierarchy
1. **Direct Matches**: The exact field name.
2. **Intent Matches**: Categories that solve the user's query.
3. **Contextual Suggestions**: Related settings (e.g., searching for "Volume" suggests "Voice Tone").

## 3. Deep Linking
Search results must be actionable. Clicking a result should:
1. Open the correct Category.
2. Scroll to the specific Field.
3. Pulse/Highlight the field for 2 seconds to guide the eye.
